# 开发加载说明

当前版本是无构建工具的 Chrome Extension Manifest V3 原型，可以直接加载项目目录。

## 依赖

项目使用 `pdfjs-dist` 在插件设置页本地解析文本型 PDF，并使用本地 Node.js + Express 后端做简历 AI 结构化和职位匹配分析。

如果重新拉取项目或 `node_modules` 不存在，先在项目根目录执行：

```sh
npm install
```

## 启动本地后端

1. 复制环境变量示例：

```sh
cp .env.example .env
```

2. 在 `.env` 中填写：

```sh
OPENAI_API_KEY=replace-with-your-api-key
OPENAI_MODEL=gpt-4o-mini
OPENAI_BASE_URL=https://api.openai.com
```

3. 启动后端：

```sh
npm run server
```

后端固定监听 `http://localhost:3000`，接口包括：

- `POST /api/resume/structure`：简历结构化。
- `POST /api/match/analyze`：职位匹配分析。

## 加载插件

1. 打开 Chrome。
2. 访问 `chrome://extensions/`。
3. 打开右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本仓库所在的项目目录。

## 验证目标岗位设置

1. 在扩展程序页面找到“求职匹配助手”。
2. 点击“详细信息”。
3. 点击“扩展程序选项”。
4. 输入目标岗位并保存。

## 验证简历上传

1. 在 Boss 页面侧边栏点击“打开设置页”，或点击浏览器工具栏里的插件图标。
2. 上传 `.txt` 简历，或上传文本型 `.pdf` 简历。
3. 检查“简历文本预览”是否出现正文。
4. 确认本地后端 `http://localhost:3000` 已启动。
5. 点击“结构化并保存简历”。
6. 检查“结构化简历 JSON（可编辑）”是否出现教育、工作、项目、技能等字段。
7. 如需修正结构化结果，编辑 JSON 后点击“保存编辑结果”。

说明：扫描版或图片型 PDF 暂不支持，需要后续接 OCR。

## 验证 AI 匹配

1. 确认本地后端 `http://localhost:3000` 已启动，且 `.env` 已配置模型 API Key、模型名和 Base URL。
2. 回到 Boss 职位页面，确认侧边栏已捕捉当前职位且简历已保存。
3. 点击“AI 分析”。
4. 侧边栏应展示匹配分数、分数拆解、匹配点、风险点、简历亮点和投递建议。
5. 分析成功后，“最近分析”应新增一条当前职位记录。

说明：简历结构化和职位匹配分析的 API Key 都放在本地后端 `.env`，插件端不再保存模型 API Key。

## 验证数据与隐私控制

1. 打开插件设置页。
2. 查看“数据与隐私”区域，确认能看到本地保存和发送给本地后端的数据说明。
3. 点击“删除简历数据”，确认简历文本和结构化简历被清空。
4. 点击“清空分析历史”，回到 Boss 页面确认“最近分析”被清空。

## 验证 Boss 页面捕捉

1. 打开 Boss 直聘职位页面。
2. 页面右侧应出现“求职匹配助手”侧边栏。
3. 侧边栏应自动显示职位、公司、薪资、标签和 JD。
4. 如果自动捕捉不完整，可以在“手动兜底”中粘贴 JD。


## 采集 Boss 回归样例

用于准备真实职位回归测试集，不需要打开 Boss 开发者工具。

1. 重新加载本地插件。
2. 打开 Boss 职位页，等侧边栏显示“已自动捕捉当前职位”。
3. 点击侧边栏里的“导出职位样例”。
4. 如果复制成功，直接把剪贴板里的 JSON 发给开发者；如果复制被浏览器拦截，插件会下载一个 `boss-job-*.json` 文件。
5. 每条样例最好补充备注：普通职位/猎头职位、肉眼正确公司名、字段是否有争议。

导出的 JSON 包含页面信息、原始 `window._jobInfo`、DOM 抽取结果、合并后结果和 expected 草稿，可后续整理进 `tests/fixtures/boss-job-info/`。

## 当前限制

- 简历结构化和职位匹配分析已通过本地后端调用 OpenAI-compatible Chat Completions 接口。
- 插件端不再保存模型 API Key。
- Boss 页面捕捉已接入结构化数据优先、DOM 兜底；普通职位整体可用，但猎头职位的委托公司字段仍需用真实样例继续验证。
- 猎头职位中，“委托公司”“招聘方/猎头公司”“工商信息公司”可能被页面混合展示，当前版本不要把委托公司作为强可信字段。
- 扫描版 PDF 暂不支持。
